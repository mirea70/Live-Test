# 요구사항

1. 목적

사용자가 짜장면 / 짬뽕 중 하나를 선택해 투표하고, 현재까지의 누적 결과를 실시간으로 확인할 수 있는 웹페이지를 구현한다.
평가 포인트는 UI 완성도가 아니라 다음을 확인하는 것이다.

서버–클라이언트 역할 분리

상태 변경(투표)과 조회(결과)의 구분

기본적인 웹 구조 이해

2. 화면 요구사항
2.1 투표 페이지

URL: http://{도메인 또는 Public IP}/vote

기능

선택지: 짜장면, 짬뽕

하나 선택 후 “투표하기” 버튼 클릭

투표 성공 시 /result로 이동

2.2 결과 페이지

URL: http://{도메인 또는 Public IP}/result

기능

현재까지의 투표 결과 표시

짜장면: N표

짬뽕: M표

다시 투표 링크 제공 (선택)

3. 데이터 처리

투표 결과는 서버에서 관리

저장 방식: 자유(DB/파일/메모리)

새로고침 시 결과 유지 필수

서버 재시작 후에도 유지되면 가산점

어뷰징 방지 기술 포함 시 가산점

4. 비기능 요구사항

결과 조회는 빠르게 응답할 것(단순 카운트)

입력 검증: 선택지는 지정된 enum만 허용

서버 오류/네트워크 오류 시 사용자에게 최소한의 안내 제공
